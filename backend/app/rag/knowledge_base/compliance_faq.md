# SentinelAI Compliance FAQ

## Data Privacy

**Q: Where is my data stored?**
A: All data is stored in SOC 2 Type II certified data centers in the US (us-east-1) and EU (eu-west-1). Enterprise customers can choose their preferred region.

**Q: Is SentinelAI GDPR compliant?**
A: Yes. We act as a Data Processor under GDPR. We offer Data Processing Agreements (DPA) for all customers. Enterprise customers receive a custom DPA.

**Q: How do I handle a Data Subject Access Request (DSAR)?**
A: Navigate to Settings → Privacy → DSAR Management. You can export all data for a specific contact or request deletion. We process DSARs within 72 hours.

**Q: Can I delete customer data?**
A: Yes. Use the "Right to Erasure" feature in Settings → Privacy. Data is permanently deleted within 30 days, including backups.

## Security

**Q: How is data encrypted?**
A: Data is encrypted at rest (AES-256) and in transit (TLS 1.3). API keys are hashed using bcrypt.

**Q: Do you have SOC 2 certification?**
A: Yes, SOC 2 Type II. Our latest audit report is available upon request for Enterprise customers.

**Q: How do you handle security incidents?**
A: We follow a documented Incident Response Plan. Customers are notified within 72 hours of a confirmed data breach per GDPR requirements. Enterprise customers receive immediate notification.

## AI & LLM

**Q: Is my email data used to train AI models?**
A: No. We use OpenAI's API with data privacy mode enabled. Your data is never used for model training. Enterprise customers can opt for a private model deployment.

**Q: How do you prevent prompt injection?**
A: We employ multiple layers: input sanitization, system prompt hardening, output validation, and anomaly detection. All inputs are screened before being sent to the LLM.

**Q: What happens if the AI makes a mistake?**
A: All AI actions are logged with full reasoning traces. High-risk actions (legal, security) always require human approval. You can configure confidence thresholds for auto-actions.

## Legal

**Q: What if we receive a legal threat via email?**
A: SentinelAI automatically detects legal threats and flags them with "do_not_auto_reply". These are escalated to your legal team. No automated responses are sent.

**Q: Who owns the AI-generated responses?**
A: You own all content generated by SentinelAI for your organization.
